# MAP_ARCHITECTURE.md — 地图三层架构设计（Phase 2 地图重构）

> **地位**：Phase 2 地图重构的技术设计源文件，实现工作单据此出。
> **建立**：2026-06-02。**关联决策**：`PRODUCT_DIRECTION.md` D5。**前置**：存档迁移框架（技术债 C-2）。

## 1. 目标
从「6 省 = 6 个色块（每省 1 格）」演进为「**省 = 3~4 格（tile）聚合的区域 + 格级地形 + 美化渲染**」，支撑硬核策略的地形 / 补给深度与可扩展的中大型地图。

## 2. 三层数据模型
```
Country(国) ─持有→ Province(省 / region) ─聚合→ Tile(格)
```
- **TileState（新增 Domain/State）**：`id, gridX, gridY, terrain, provinceId(归属省)`；可选 `elevation, resource`。
- **ProvinceState（改造）**：去掉"自身即一格"假设；新增 `tileIds: List<string>`；地图坐标语义从省下沉到格。产出/基建仍可省级持有；**地形从省级移到格级**。保留 `owner/controller/resistance/compliance/isCapital/victoryPoint`。
- **CountryState**：不变（持 provinceIds 经 unitIds/commanderIds 等）。
- **过渡兼容**：现有 6 省 → 每省细分为 3~4 格起步（**是细化，不是推倒**）。

## 3. 邻接：从手写到自动推导
- 现状：省级 `neighbors` 手写数组（不可扩展、易漂移）。
- 目标：邻接由**格级相邻**自动推导：
  - 格邻接：网格坐标相邻（先做 4 邻，地图风格定后可选 6 邻 hex）。
  - 省邻接：∃ A 省某格与 B 省某格相邻 ⇒ A、B 省邻接（启动时算一次 + 缓存，占领变更时增量更新）。
- 收益：地图扩张**零手写邻接**；根除"手写邻接漂移"老问题。

## 4. 粒度抉择（关键设计决策）
| 维度 | 粒度 | 理由 |
|---|---|---|
| 战斗 / 占领 / 驻军 | **省级** | 回合制；格级会让微操爆炸、不符硬核回合节奏 |
| 移动 | **省级**（经省邻接） | 与战斗一致 |
| 地形 | **格级** | 地形是省内地貌，驱动呈现与补给 |
| 补给 / 补给路径 | 走格、判定落省 | BFS 沿格（地形影响通行成本），切断/包围判定仍以省为单位 |

> 一句话：**决策在省、地貌在格**。战斗**复用现有省级 BattleResolver**；省级地形修正由省内格地形聚合得出（如"省内山地格占比 → 防御加成"或"主导地形决定"——具体公式 P2.4 设计时定，数值入 config 规则 5）。

## 5. 渲染换栈
- 现状：UI Toolkit 绝对定位色块（呈现层反复救火：黑屏 / 文字重叠 / 深色字）。
- 目标：**世界空间 Tilemap（`com.unity.2d.tilemap` 已在 manifest）+ 正交相机**渲染地图；**UI Toolkit 退回只做 HUD / 面板浮层**。
- 交互改造：选省从 UI 事件 → 相机射线 / Tilemap 命中 → 映射 tile → province → 复用现有 `SelectProvince`。
- 收益：地形贴图 / 高度 / 美化 / 缩放平移成为可能；呈现与逻辑彻底分离。
- 风险：呈现架构一次性较大调整，需 PlayMode 重新验证（相机 / 输入 / 命中），延续"Unity 呈现层坑多"的经验，预留 Claude 手修 SetupScene 的余量。

## 6. 借这次重构还两笔债
- **C-2 存档迁移（前置 · 必做）**：三层模型必然改变存档结构（province 加 tileIds、新增 tiles 集合）。**重构前先建迁移框架**：`schemaVersion` 路由 + 旧档（每省 1 格）→ 新档（每省 N 格）迁移器。不再裸靠 Newtonsoft 容错。**这是把 C-2 从"上线红线"提前偿还的最佳时机。**
- **C-5 空间索引（随扩容做）**：格 / 省一多，`ReadModelBuilder` 的 O(P×U) 全遍历会爆。改为按 `provinceId` 索引部队（`Dictionary<province, List<unit>>`），驻军查询 O(1)。

## 7. 地图生产管线（建议）
几十~上百格不可能手编 JSON：
- **(A · 建议先做)** 轻量**地图编辑器**（Unity Editor 工具）：刷格 / 刷地形 / 划省 / 导出 JSON。配置工具链 C-3 的延伸。
- **(B)** 程序化生成（噪声地形 + 省划分）——更大世界 / 后期再做。
- Phase 2 先 A（可控、够用），B 留更大世界。

## 8. 实现顺序（详见 PHASE2_ROADMAP.md）
1. **数据层**（P2.2）：TileState + Province.tileIds + 邻接推导 + 存档迁移（C-2）。
2. **渲染层**（P2.3）：Tilemap + 相机 + 选省命中；UI 退 HUD。
3. **玩法层**（P2.4）：地形落格、补给走格、地形修正接入省级战斗。
4. **内容层**（P2.5）：扩到中等地图（~20–40 省）+ 空间索引（C-5）+ 地图编辑器。
