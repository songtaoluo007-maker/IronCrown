## 0. 决策纪要（2026-06-02 人类拍板 · 规则 14）
本轮战略讨论起因：Phase 1（C4→C17）同时服务了**硬核 wargame** 与**抽卡手游**两个互斥用户画像，定位拍板以消除冲突。

| 编号 | 决策 | 含义 |
|---|---|---|
| **D1** | 目标用户 = **硬核策略玩家为主** | 重推演 / 确定性 / 技巧成长 / 长局；最恨 P2W |
| **D2** | 商业模型 = **免费 + 内购 · 服务型**（F2P / LiveOps） | 比单机大一个量级：需账号/服务端/远程配置/埋点 |
| **D3** | 变现红线 = **绝不卖战力** | 只卖时间 / 外观 / 内容 / 便利 |
| **D4** | **取消抽卡（gacha）变现机制** | C16/C17 付费抽卡退役；将军转养成（横向特化、战功解锁、不卖战力） |
| **D5** | 地图演进 = **省由 3~4 格（tile）聚合 + 地形 + 美化** | 世界空间 Tilemap；详见 `MAP_ARCHITECTURE.md` |

## 1. 目标用户（D1）
- **主**：硬核策略玩家（HoI4 / EU4 / P 社受众）。要确定性、深推演、技巧型成长、单局可投入长时间。
- **反需求（必须避开）**：pay-to-win、数值碾压、抽卡黑箱、逼氪墙。该群体对此**零容忍**——是差评与流失的头号来源。
- **外围**：轻策略玩家——用可调难度 + 新手引导承接，但**不为他们牺牲核心深度**。

## 2. 商业模型（D2）
- 形态：免费下载 + 内购 + 长期运营（赛季 / 活动 / 版本更新）。
- 这是对**服务型工程**的承诺：账号系统、服务端、远程配置（数值热更）、埋点分析四件套（Phase 1 全无；见 `PHASE2_ROADMAP.md`，Phase 3 占位，Phase 2 起做埋点雏形）。
- 原则：**留存 > 变现**。硬核 F2P 先靠内容与深度留人，再靠外观/通行证变现。

## 3. 变现红线与允许清单（D3）
**绝不卖（伤平衡 = 伤核心用户）**：
- 战力数值（更强的卡/兵/buff）、付费抽卡概率、付费加速战力成长、复活续命、付费独占强力单位。

**允许卖（不影响平衡）**：
- **赛季通行证**：外观 + 游戏内货币 + 非战力便利（留存 + 变现主引擎）
- **外观**：将军立绘皮肤 / 地图主题 / 单位涂装 / UI 皮肤
- **内容 DLC**（P 社模式）：新战役 / 阵营·国家包 / 新地图
- **便利非战力**：额外存档位 / 回放导出 / 深度统计面板 / 自定义房间
- **支持型**：Supporter Pack / 捐赠

**判定准则**：若一个付费项能让两个同水平玩家分出"花钱方更容易赢" → 违红线，否决。

## 4. 抽卡退役 + 将军转养成（D4）
- **取消**：gacha 抽卡（`DrawCard` / 保底 / 升星变现）、商城买券（C16/C17 变现本体）。
- **保留并转型**：将军 = 养成系统，但
  - **获取**靠游戏内进度（战功 / 成就 / 战役解锁），不靠付费抽卡；
  - **强度横向特化**（山地 / 突击 / 防守 / 后勤专长 = 策略选择），**非纵向碾压**——SSR = 更特化、更挑场景，不是更强；
  - `CommanderState.starLevel` / 卡池数据结构可复用为养成底座，仅移除付费入口与随机抽取。
- **执行**：见 `PHASE2_ROADMAP.md` P2.1（专项工作单退役/改造）。**Phase 1 先按现状收口合入**，避免与正在进行的 `Phase1-closeout-fix` 冲突。

## 5. 北极星与度量方向
- 主：**留存**（D1/D7/D30）+ 单局时长 / 完成率（硬核手感代理指标）。
- 变现健康：付费转化 + ARPPU，但以**外观渗透率**为健康指标（非战力付费占比应趋零）。
- 依赖埋点（Phase 2 起雏形）。

## 6. 与工程宪法的关系
- 本文件管"做什么、给谁、靠什么活"；`PROJECT_RULES.md` 管"怎么做"。冲突时：产品方向定目标、工程宪法定手段，二者均不可违。
- **确定性架构**（整数 + SplitMix64）在本定位下升级为**核心资产**——异步 PvP / 回放 / 反作弊的地基（见 `ARCHITECTURE.md` 附录 C）。Phase 1 看似"过度"的确定性投资，在 F2P 服务型 + 硬核 + 未来 PvP 下回本。
