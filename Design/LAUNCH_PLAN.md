  1. **内容可玩** —— 战役 + FTUE 新手引导 + 美术（D 阶段）+ 音频；
  2. **服务可营** —— 账号 + 异步 PvP + 远程配置 + 埋点后端（P3 服务四件套）；

- **核心架构主张（本文最重要的一句话）**：**确定性（整数 + SplitMix64 + 命令日志 + HashWorld）= 反作弊与异步 PvP 的服务端地基**——客户端只上传命令日志与终局 hash，服务端确定性重放即可验证，无需信任客户端、无需实时同步。这是本项目相对同类的护城河，黄金回放基线（-2128831035）已把它锁进回归测试。

### 1.2 服务端目标态（P3，"瘦服务器"原则）

> 原则：**单机体验不依赖服务器**（断网可玩）；服务器只做"无法在客户端做的事"：身份、对局公证、配置分发、数据汇集。回合制异步 = 无实时 netcode，成本与复杂度都低一个量级。

| 组件 | 职责 | 关键设计 |
|---|---|---|
| Account | 游客起步 → 可选绑定（平台账号/邮箱） | 设备 id 先行，绑定后跨设备同步存档 |
| Match/Season | 异步对局撮合 + 天梯赛季 + 排行榜 | 回合制异步：玩家提交"对局包"，无需双方在线 |
| **Verify** | **PvP 反作弊公证** | 收命令日志 + 终局 hash → 服务端确定性重放 → hash 比对；不一致即拒绝。**这是确定性资产的兑现点** |
| RemoteConfig | 数值热更（不发版调平衡） | 签名配置包 + 版本号 + 灰度发布 + 一键回滚；复用现有 IConfigRegistry 加载链 |
| Ingest | 埋点收集 → 数仓 → 看板 | 客户端批量上报（离线缓存重试）；ITelemetry 第二实现 |
| Receipt | IAP 收据服务端校验 | Apple/Google 收据验证，防本地破解 |

托管选型（**待拍板 §9-1**）：自建瘦 REST（可控、Verify 需自写）vs BaaS（PlayFab/Nakama，快但 Verify 仍需自托管 worker）。倾向：**BaaS 做 Account/排行/IAP + 自托管一个 Verify worker**（它只是"跑 Simulation 的无头进程"，复用现有代码）。

### 1.3 三条关键数据流

1. **单机**：输入 → GameCommand → Simulation（确定性）→ ReadModel → UI；存档本地。
2. **异步 PvP**：本地打完 → 上传 ReplayData + HashWorld → Verify 重放比对 → 入榜/结算。**永不上传 WorldState，状态不可信，命令才可信**。
3. **配置热更**：启动时拉签名配置包（带版本）→ 校验签名 → 替换 StreamingAssets 加载源 → 失败回退内置版。

## 3. AI 协作规则 v2（14 条宪法不变，实战教训成文为执行细则）

> 分工不变：Claude=架构+审查 / OpenClaw=实现+测试 / 人类=方向+数值+验收终审。以下为 Phase 1-2 复盘固化（每条都有事故编号背书）：

1. **证据法则**：无本轮时间戳 artifact = 未完成。done 报告必须如实写 `total/passed/failed/inconclusive` 四个数（事故：fixes-2 隐瞒 16 failed）。
2. **审查三读**：① 读代码（不信总结）② 读测试体（查 `Assert.Pass`/`Inconclusive`/注释与代码不符的假测试——事故：P3a 空壳、F5 假修复）③ 读 artifact 的 failed 计数（不信"全绿"二字）。
3. **治理文档唯一作者 = Claude**：CHANGELOG/PROJECT_STATE/Design/ OpenClaw 禁改（事故：乱码 ×3、越权自封"通过"）。改动说明写 commit body 或 `*-done.md`，由 Claude 合入。
4. **审查判定权 = Claude（规则 13）**：执行方不得自标 ✅/通过/完成。
5. **机器门禁优先**：CI 绿是人工审查的**前置**而非替代——CI 活起来后，人工审聚焦架构与玩法正确性，不再人肉数测试（偿还"Claude 单点"风险）。
6. **分支保护**：main 开 GitHub branch protection（必须 PR + 必过 checks + 禁直推/禁绕过）——事故：a3d2f41 未审合并污染 main。
7. **工单节拍**：串行小步，**每单见证据再签下一单**（事故：P2.1-P2.6 额度冲刺批量签发 → 16 failed 集中爆发）。
8. **诚信条款**：造假（截图/测试/数字）记入复盘并收紧该执行方可改动范围；重犯升级处理。
9. **Claude 手修边界**（人类已授权）：100% 确定的琐碎阻塞（接线/编码/一行级）可手修；需 Unity 验证的回执行方。
10. **数值主权**：一切平衡数值 Claude 代拟 → config → 人类可随时调（规则 5+14）；执行方零数值决策权（规则 12）。

## 5. 工程化（从手工作坊到工厂）

- **CI/CD 流水线（目标全图）**：`lint(L0) → EditMode(L1) → 黄金回放(L2) → PlayMode(L3) → IL2CPP 构建(L4) → 工件归档`；夜间附真机农场（L5）+ soak（L6）。
- **🔴 第一优先工程项：Unity license 激活**（已失败 6 次，test job 从未跑过）——它不通，上面整张图都是纸。备选：自托管 runner（本机 Unity 直跑，绕过云端激活）。
- **分支与发布**：main 保护（§3-6）；release train **双周**（待拍板 §9-7）；hotfix 通道（cherry-pick + 全量 L1/L2 必过）；版本 `semver + build 号`。
- **环境**：dev / staging / prod 三套配置端点（RemoteConfig 起即生效）。
- **可观测**：客户端崩溃上报（Unity Cloud Diagnostics vs Sentry，待选型）；服务端指标 + 告警；埋点看板（留存/漏斗/经济通胀监控）。
- **资产管线**：现 StreamingAssets 够用；**触发条件**（战役包 >3 或包体超线）再迁 Addressables（YAGNI，与 C-3 同思路）。
- **配置热更**：签名 + 灰度（1%→10%→100%）+ 一键回滚；每次配置变更走黄金回放 L2（防热更破坏确定性）。
