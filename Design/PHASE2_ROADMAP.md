# PHASE2_ROADMAP.md — Phase 2 路线图

> **地位**：Phase 2 执行总纲。**关联**：`PRODUCT_DIRECTION.md`（为何做）+ `MAP_ARCHITECTURE.md`（地图怎么做）。
> **建立**：2026-06-02。每子阶段审查通过后更新本文件状态 + PROJECT_STATE。

## Phase 2 主题
把 Phase 1 堆出的军事系统,放到一张**有地形、能美化、可扩展的多格地图**上验证硬核手感;同时补齐工程地基(CI / 存档迁移 / 埋点雏形),为 Phase 3 服务型运营铺路。

**一句话:Phase 1 证明"系统能跑",Phase 2 证明"硬核好玩"。**

## 指导原则
1. **先骨架后收窄**(项目一贯气质):每子阶段先最小可玩,验证后再加内容。
2. **先验证硬核手感,再叠运营/变现**——不给"还没人留下来"的游戏做商业化。
3. **重构必还债**:地图重构同时偿还 C-2(存档迁移)、C-5(空间索引)。
4. **不卖战力红线贯穿**(PRODUCT_DIRECTION D3)。

## P2.1 抽卡退役 + 将军转养成（执行 D4）
- **目标**:移除 gacha 付费抽卡 / 商城买券变现;将军获取改为战功 / 成就 / 战役解锁;强度横向特化。
- **关键**:移除 `DrawCard` 付费路径 / 保底 / 商城券;**保留** `CommanderState`/技能/`starLevel` 作养成底座;新增"战功→解锁"通道;校正卡强度为横向(改 `commanders.json` 数值 + `CommanderSkillEvaluator` 语义,规则 5/14);`GachaPanel`/`ShopPanel` 改造为"将军图鉴/养成页"或下线。
- **验收**:无任何付费抽卡入口;将军靠游戏内进度获得;同水平对局无"抽到强卡碾压"。

## P2.2 地图数据层三层重构
- **目标**:`TileState` + `Province.tileIds` + 邻接自动推导(MAP_ARCHITECTURE §2–3)。
- **关键**:新增 `TileState`;`ProvinceState` 加 `tileIds`;邻接由格推导;`SaveMapper/SaveModels` 纳入 tiles 并经 C-2 迁移框架处理旧档(6 省→每省 3–4 格)。
- **验收**:存读档等价(含 tiles);自动邻接与手写版回归一致;6 省细分为 ~18–24 格仍跑通既有玩法。

## P2.3 地图渲染换栈
- **目标**:世界空间 Tilemap + 正交相机;UI Toolkit 退 HUD 浮层(MAP §5)。
- **关键**:Tilemap 渲染省 / 格;相机射线选省 → `SelectProvince`;地形贴图;缩放平移。
- **验收**:点击选省正确;PlayMode 通过(相机 / 输入 / 命中);地图可视明显优于色块。

## P2.4 地形玩法落地
- **目标**:地形从格级真正影响玩法。
- **关键**:地形修正接入**省级**战斗(省内格地形聚合公式,数值入 config);补给 BFS 走格、地形影响通行成本;移动/补给路径体现地形。
- **验收**:山地省防御显著高于平原(可感知);补给路径绕地形;数值人类可调(规则 14)。

## P2.5 中等地图内容 + 性能
- **目标**:扩到 ~20–40 省可玩战役 + 偿还 C-5。
- **关键**:**地图编辑器**(MAP §7-A)产出中等地图;**空间索引**(按 `provinceId` 索引部队)替换 O(P×U);一场有起承转合的战役场景。
- **验收**:中等地图回合结算流畅(profiling 达"轻量移动端"目标);一局完整可玩。

## P2.6 硬核手感验证（垂直切片 + 埋点雏形）
- **目标**:真人完整玩一局中等地图战役,用数据验证方向。
- **关键**:埋点雏形(关卡流失 / 单局时长 / 操作热点,本地或轻服务端);收集硬核玩家反馈。
- **验收**:真人垂直切片报告 + 埋点数据 + 下阶段取舍结论。

## Phase 3+ 占位（不在 Phase 2 做,先占位防架构走偏）
- **服务型四件套**:账号系统、服务端、远程配置(数值热更)、完整埋点分析。
- **异步 PvP / 天梯 / 排行**——确定性回放地基的兑现(核心资产回本处)。
- **变现实装**:赛季通行证、外观商店、内容 DLC(框架先于实装)。
- **LiveOps**:活动日历、赛季机制、运营后台。
- **新手引导 / 本地化(C-6)**。

## 明确不做（Phase 2 YAGNI）
- 实时联机(确定性天然支持**异步**,不做实时)。
- 变现实装(P2 只定原则 + 留接口,不接支付)。
- 完整 LiveOps / 重服务端(P2 埋点用最轻方案)。
- 3D(项目定性 2D)。

## 风险与依赖
- **渲染换栈**是呈现层大改,PlayMode 验证坑多(历史经验),预留 Claude 手修 SetupScene 余量。
- **迁移框架**若 P2.0 没筑好,P2.2 会被迫裸改存档(重蹈覆辙)。
- **范围膨胀**:硬核深度 + 大地图 + 服务型任一都大;严守"先中等地图可玩,再叠"。
- **数值平衡**:地形 / 将军横向化需平衡框架(power budget),否则手调爆炸。
