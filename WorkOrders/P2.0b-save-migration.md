# P2.0b — 存档迁移框架（schemaVersion 路由 + 迁移器 + 旧档测试）

## 地位 / 背景
Phase 2 **P2.0 前置**。地图三层重构（P2.2，见 `Design/MAP_ARCHITECTURE.md`）将改变存档结构（`Province` 加 `tileIds`、新增 `Tile` 集合），**必破旧档**。当前 `GameState` 连 `schemaVersion` 都没有，全靠 Newtonsoft 容错。本单建立**迁移框架**——偿还技术债 **C-2**，作为 P2.2 的硬前置。

执行：OpenClaw。分支：`feature/p2.0-foundation`。

> ⚠️ 本轮起仓库已上 **CI 门禁**（`.github/workflows/ci.yml`）：push 后 **lint job（分层+UTF-8）必过**；若 `UNITY_LICENSE` 已配，**test job（EditMode+PlayMode）也必须绿**。CI 会自动拦"测试没真跑/分层违规/编码乱码"——**不要再靠口头声明,以 CI 绿 + artifact 为准**。

## 范围
1. **`GameState` 加 `schemaVersion`**（当前缺失）：`public int schemaVersion;`;在 Application 定义常量 `SaveSchema.CURRENT = 1`;`SaveMapper.ToSave` 写入 `CURRENT`。
2. **迁移接口** `Application/Persistence/ISaveMigration.cs`：
   ```csharp
   public interface ISaveMigration {
       int FromVersion { get; }              // 适用于该版本 → 升到 FromVersion+1
       Newtonsoft.Json.Linq.JObject Migrate(Newtonsoft.Json.Linq.JObject raw);
   }
   ```
3. **Runner** `Application/Persistence/SaveMigrationRunner.cs`：持有迁移器列表（按 `FromVersion` 升序）;入口 `JObject Upgrade(JObject raw)`：读 `raw["schemaVersion"]`（缺则默认 1）→ 当 version < `CURRENT` 时链式调用对应迁移器 → 每步把 `schemaVersion` +1 → 返回升级后 JObject。
4. **Load 流程接入**（改 `Infrastructure` 的 SaveRepository 或 `GameSessionService.Load`，**保持 `ISaveRepository` 接口不变**）：读档时 `JObject.Parse(json)` → `runner.Upgrade(raw)` → `raw.ToObject<GameState>()` → `SaveMapper.ToRuntime`。
5. **样例迁移器**：写一个占位迁移器（如 `Migration_0to1`：给缺字段的最老档补 `schemaVersion=1`），证明管线工作。**真实地图迁移器（每省 1 格 → 多格）留 P2.2,本单不写。**
