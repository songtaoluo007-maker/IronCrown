# P2.6 — 埋点雏形（硬核手感验证的基础设施）

## 地位 / 依赖
**前置:P2.5 通过**(需中等地图可玩)。对应 `Design/PHASE2_ROADMAP.md` P2.6。

> ⚠️ **范围切分**:P2.6 的「硬核手感验证」**主体是真人 Play + Claude 分析**(规则 14,人类终审体验)。**OpenClaw 只做埋点代码基础设施**,不负责"验证结论"。本单只签发埋点部分。

## 范围（OpenClaw 做）
**轻量本地埋点**,记录可度量信号供后续分析:
1. 新 `Infrastructure/Telemetry/ITelemetry.cs` + 本地实现(写本地 JSON/CSV,**不接外部服务/网络** — 服务端是 Phase 3):
   - 事件:`turn_advanced`(回合号/各国资源快照)、`battle_resolved`(攻防/结果)、`province_occupied`、`commander_unlocked`、`game_over`(胜负/总回合/时长)、`command_issued`(命令类型计数 = 操作热点)。
2. 接入点:`GameSessionService`/`TurnResolver` 关键节点发埋点事件(经 `ITelemetry`,**Application 层注入,不污染 Simulation 纯函数**)。
3. 单局汇总:一局结束输出 `Design/telemetry/session-{time}.json`(单局时长、总回合、关键漏斗:发展→首战→首占→终局)。

## 数值 / 设计（Claude 代拟）
- 埋点**只读不改玩法**:纯旁路记录,绝不影响确定性/数值。
- 字段轻量(避免性能负担);本地文件,无网络。

## 文件清单
- 新 `Infrastructure/Telemetry/ITelemetry.cs`(接口,放 Contracts 或 Application 端口) + 本地实现(Infrastructure)
- `Application/Session/GameSessionService.cs` / `Simulation/TurnResolver.cs` 关键节点接入(经注入端口,核心层不依赖 Telemetry 实现)
- DI 注册(`GameLifetimeScope`)
- 测试:埋点事件触发 + 字段正确(用 stub ITelemetry 断言被调)
