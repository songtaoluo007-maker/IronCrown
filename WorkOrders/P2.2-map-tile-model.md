# P2.2 — 地图数据层三层重构（Tile / Province / 邻接自动）

## 地位 / 依赖
**前置:P2.0b 存档迁移必须先通过 CI**(本单要 bump schema 到 2 + 写 Migration_1to2,依赖迁移框架)。对应 `Design/MAP_ARCHITECTURE.md` §8 实现顺序**第 1 步(数据层)**。后续 P2.3/P2.4/P2.5 依赖本单。

## 范围（严格照 MAP_ARCHITECTURE §2–3）
**纯数据层重构,不碰渲染/玩法**。把"省=1 格色块"升为"省=多格(tile)聚合 + 邻接自动推导"。

1. **`TileState`(新 `Domain/State/TileState.cs`)**:`id, gridX, gridY, terrain, provinceId`(归属省)。
2. **`ProvinceState`**:新增 `List<string> tileIds`。地图坐标语义下沉到格;省保留 owner/controller/resistance 等。
3. **邻接自动推导(新 `Simulation/AdjacencyResolver.cs`)**:
   - 格邻接:网格坐标 4 邻。
   - 省邻接:∃ A 省格与 B 省格相邻 ⇒ A、B 省邻接;启动算一次 + 缓存,占领变更不影响(邻接是地理静态)。
   - **替代手写 `ProvinceState.neighbors`**(改由推导填充,或保留 neighbors 字段但由 Adjacency 计算写入)。
4. **存档(经 P2.0b 迁移框架)**:
   - `SaveModels`:新增 `TileSaveData`;`ProvinceSaveData` 加 `tileIds`;`SaveSchema.CURRENT = 2`。
   - `SaveMapper`:tiles 双向持久化 + province.tileIds。
   - **`Migration_1to2`(新)**:v1 旧档(省无 tiles)→ 按省 `gridX/gridY` 程序生成默认格 + 回填 tileIds,`schemaVersion→2`。
5. **`WorldInitializer`**:从 config 生成 tiles(每省细分,见下数值)。

## 数值（Claude 代拟,人类可调）
- **6 省每省细分为 4 格**(2×2 子网格;格坐标 = 省 gridX*2+{0,1}, gridY*2+{0,1})。共 24 格。
- **P2.2 阶段格地形 = 继承省主地形**(格级地形差异化留 P2.4,本单不做地形玩法)。
- tile id 规约:`{provinceId}_t{n}`(n=0..3),确定性。

- `TileModelTests`:每省 4 格生成、tile id 确定性、province.tileIds 正确。
- `AdjacencyResolverTests`:**自动推导的省邻接 == 现有 6 省手写 neighbors**(关键回归 — 不能改变邻接关系)。
- `SaveLoadEquivalenceTests` 扩:含 tiles 的世界存读 hash 等价;`Migration_1to2` 把 v1 档升 v2 且 tiles 正确生成(用 P2.0b 测试模式)。
