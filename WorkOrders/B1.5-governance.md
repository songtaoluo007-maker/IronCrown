## 0. 目标与范围判断
让玩家用两个经营杠杆影响国家：**税率档**（低/中/高）影响税收与稳定；**民生档**（紧缩/正常/宽裕）影响民生支出与稳定。立刻可感知（调高税→国库涨、稳定跌）。

**刻意不做「生产分配」**：当前无军队，装备产线分配无消耗端、玩家无法感知价值——为做而做只增复杂度（规则 9，不过度设计）。生产分配推迟到有军事系统之后（B3 后）。本单 = 税率 + 民生两个档位命令。

## Phase 0
- 新建分支 `feature/b1.5-governance`；起点全套测试全绿。
- 不直接编辑 `CHANGELOG.md`（写 PR 描述，Claude 合入）；UTF-8 保存。

## 1. 数值（Claude 已写入 economy.json，勿改）
`economy.json` 的 `global` 行已新增（索引 0=低/紧缩，1=中/正常，2=高/宽裕）：
- `taxRatePercents`: `[70,100,130]` —— 税收倍率（百分比）
- `taxStabilityDeltas`: `[1,0,-2]` —— 每回合稳定修正
- `civilExpensePercents`: `[50,100,150]` —— 民生支出倍率
- `civilStabilityDeltas`: `[-2,0,2]` —— 每回合稳定修正
> 默认档位 = 1（中/正常）。OpenClaw 勿改这些数字。

## 2. 架构决策（写死）
1. **档位是玩家可改的运行时状态**，存于 `CountryState`：`taxLevel`/`civilLevel`，默认 `1`。
2. **命令复用 B1 管线**：`CommandType` 加 `SetTaxLevel`/`SetCivilLevel`；`GameCommand` 加 `int level` 字段（建造命令忽略它）。
3. **效果分工**：税收倍率/民生倍率在 `EconomyResolver.ResolveEconomy`（已注入 config）；稳定修正在 `PoliticsResolver.ResolvePolitics`（**本单给 PoliticsResolver 注入 `IConfigRegistry`**，保持"稳定归政治"语义整洁）。
4. **整数运算**（确定性）：倍率用 `值 * percent / 100` 截断，无 float 引入。

## 3. 实现规格

### 3.1 Domain
- `CountryState` 新增：`public int taxLevel = 1;` `public int civilLevel = 1;`
- `EconomyConfig`（`Domain/Config/EconomyConfig.cs`）新增：`public int[] taxRatePercents;` `public int[] taxStabilityDeltas;` `public int[] civilExpensePercents;` `public int[] civilStabilityDeltas;`

### 3.2 Contracts
- `CommandType` 加 `SetTaxLevel`、`SetCivilLevel`。
- `GameCommand` 加 `public int level;`
- `CountryView` 加 `public int taxLevel;` `public int civilLevel;`（HUD 显示当前档）。`ReadModelBuilder` 填充。

### 3.3 Application — IssueCommand 两个新 case
```
case SetTaxLevel:
    if (cmd.level < 0 || cmd.level > 2) return Reject("档位越界");
    country.taxLevel = cmd.level; return Accept();
case SetCivilLevel:
    if (cmd.level < 0 || cmd.level > 2) return Reject("档位越界");
    country.civilLevel = cmd.level; return Accept();
```
（仍先校验 `cmd.countryId == _playerCountryId`，复用现有前置校验）

### 3.4 Simulation
- `EconomyResolver.ResolveEconomy`：
  ```
  int taxLv = Clamp(country.taxLevel, 0, 2);
  int civLv = Clamp(country.civilLevel, 0, 2);
  int baseTax = (int)(country.taxIncome * stabilityMod);
  result.taxIncome = baseTax * eco.taxRatePercents[taxLv] / 100;          // 截断
  ...
  result.civilExpense = country.civilExpense * eco.civilExpensePercents[civLv] / 100;
  ```
  其余（tradeIncome/维护/通胀/netIncome/treasury）公式**不变**。
- `PoliticsResolver`：构造注入 `IConfigRegistry _config`。`ResolvePolitics` 的稳定结算中追加：
  ```
  var eco = _config.Get<EconomyConfig>("global");
  if (eco != null) {
      change += eco.taxStabilityDeltas[Clamp(country.taxLevel,0,2)];
      change += eco.civilStabilityDeltas[Clamp(country.civilLevel,0,2)];
  }
  ```
  （加进现有 `ResolveStability` 的 `change` 累加，最终 Clamp 0-100。现有腐败/政策修正保留）

### 3.5 接线（注意影响面）
- `PoliticsResolver` 构造签名变了 → 更新：`GameLifetimeScope` 注册（VContainer 自动注入 `IConfigRegistry`）；**所有 `new PoliticsResolver()` 改为 `new PoliticsResolver(config)`**（`GameSessionServiceTests`/`SaveLoadEquivalenceTests`/`DeterminismTests` 等测试里都有，逐一改）。

### 3.6 存档
- `CountrySaveData` 加 `taxLevel`/`civilLevel`；`SaveMapper` 双向映射。
- `SaveLoadEquivalenceTests`：续跑等价需覆盖"改税率档→存→读→续跑一致"。

### 3.7 Presentation
- HUD 命令栏加：税率档显示 + 调节（如「税率: 中 [-][+]」或低/中/高三按钮）；民生档同理。`MainHudController` 加 `SetTax(int level)`/`SetCivil(int level)`（与按钮回调共用、可测，沿用 `Advance()`/`BuildCivilian()` 模式）。
- 玩家国行或顶栏显示当前两档（来自 `CountryView.taxLevel/civilLevel`）。
- 修 B1 遗留：`MainHudController.Unbind()` 的 `UnregisterCallback` 用了新 lambda（无效），把回调存为字段后 Register/Unregister 同一引用。

## 4. 测试（规则 6）
- **EditMode 经济**：`taxLevel=2`（高）时 `taxIncome = baseTax*130/100`、`taxLevel=0` 时 `*70/100`（手算期望值，截断）。
- **EditMode 政治**：高税档使 `stability` 每回合 -2（取自配置）；紧缩民生 -2；用配置值断言、可改配置验联动。
- **EditMode 命令**：`SetTaxLevel(level=2)` 接受并改 `country.taxLevel`；`level=3` 被拒；非玩家国被拒。
- **续跑等价**：`SaveLoadEquivalenceTests` 加"改档→存→读→再跑"等价。
- **ConfigValidationTests**：4 个新数组存在且**长度==3**、百分比≥0。
