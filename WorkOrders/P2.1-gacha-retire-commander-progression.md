# P2.1 — 抽卡退役 + 将军战功解锁养成（执行 D4）

## 地位 / 依赖
执行 `Design/PRODUCT_DIRECTION.md` **D4**。**独立**于地图系列,可在 P2.0b 通过后任意时点做。偿还技术债 **C-7**。

## 背景
定位锁定为**硬核策略 F2P · 绝不卖战力**(D1/D3)。Phase 1 的 gacha(随机付费抽卡 + 保底 + 买券商城)是典型 P2W,与核心用户冲突。本单**退役随机抽卡变现**,将军改为**战功点定向解锁 + 横向特化**养成。

## 设计（Claude 代拟,人类可调 — 规则 14）
**核心机制:战功点(meritPoints)定向解锁**
- **货币**:复用现有 `CountryState.gachaTickets` 字段,**语义改为"战功点"**(字段名暂留以**不破存档结构**;UI/注释一律称"战功点")。`gachaPityCounter`(保底计数)**废弃**(随机抽卡没了,保底无意义)——字段保留但不再读写。
- **获取**(复用现有 `AwardTicketsForVictory` 逻辑,改名 `AwardMerit`):战斗胜利 +1 / 包围歼灭 +3 / 占领首都 +10(数值沿用现有 economy.json,不变)。
- **解锁(替代随机抽卡)**:新增 `UnlockCommander(country, world, config, cardId)`——花战功点**定向解锁玩家指定的将军**(非随机)。成本按稀有度。已拥有该卡 → **升星**(消耗同稀有度成本,直到 `maxStarLevel`)。
- **移除**:`DrawCard`(随机)、SSR 保底、`ShopResolver` 的买券(`BuyBundle`/`BuySsrTicket` 随机性部分)。`BuySpecificCardTicket` 本就是定向获取 → 并入 `UnlockCommander`。
- **横向特化**:`generalCards.json` 经 C15b 已是技能差异化、SSR 攻击 ≤+20(横向)。**本单确认无纯数值碾压卡**;若发现某 SSR 仅靠高 baseAttack/Defense 取胜(非特化技能),下调其裸数值、强化技能。**不新增数值膨胀**。

### 解锁成本（economy.json,Claude 代拟,人类可调）
```json
{
  "meritUnlockCostN":   10,
  "meritUnlockCostR":   30,
  "meritUnlockCostSR":  80,
  "meritUnlockCostSSR": 200,
  "meritStarUpMultiplier": 100
}
```
> 升星成本 = 解锁成本 × (当前星级+1) × `meritStarUpMultiplier` / 100(即每升一星价递增)。例:SSR 0→1 星 = 200,1→2 星 = 400…(人类可调曲线)。
