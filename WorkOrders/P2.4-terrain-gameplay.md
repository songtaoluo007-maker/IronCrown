# P2.4 — 地形玩法落地（格级地形影响战斗 / 补给）

## 地位 / 依赖
**前置:P2.3 通过**(渲染层已能显示格)。对应 `Design/MAP_ARCHITECTURE.md` §4 + §8 第 3 步(玩法层)。

## 范围（照 MAP §4「决策在省、地貌在格」）
让 P2.2 引入的**格级地形**真正影响玩法,但**战斗/占领仍省级**(粒度不变)。
1. **格地形差异化**:P2.2 阶段格地形继承省主地形;本单让 `WorldInitializer`/config 给省内各格**赋不同地形**(如一省含 3 平原+1 山地)。
2. **省级战斗地形 = 省内格地形聚合**:`BattleResolver` 的地形防御修正不再用单一省 terrain,改由**省内格地形聚合**得出。
3. **补给走格**:`SupplyResolver` BFS 沿**格**(地形影响通行成本),切断/包围判定仍落省(MAP §4)。
4. **地形倍率移 config**(消除硬编码):`BattleResolver.GetTerrainDefenseMultiplierInt` 的 switch 硬编码值 → 移入 config(规则 5)。

## 数值（Claude 代拟,人类可调 — 规则 14）
- **聚合方式 = 省内主导地形**(省内出现最多的格地形作为该省战斗地形;平票取防御倍率高者)。**人类可改为加权平均**(注释标注两种方案)。
- **地形防御倍率(移入 `economy.json`,沿用现有 BattleResolver 数值)**:
  ```json
  "terrainDefenseMult": { "Plain":100,"Forest":110,"Hills":115,"Mountain":125,"Urban":130,"Swamp":120,"River":120,"Coastline":105 }
  ```
- **补给通行成本(格级,代拟)**:Plain 1 / Forest 2 / Hills 2 / Mountain 3 / Swamp 3 / River 2 / Urban 1 / Coastline 1。BFS 累计成本超阈值视为补给衰减(阈值代拟,人类可调)。
