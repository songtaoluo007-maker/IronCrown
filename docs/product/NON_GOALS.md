# IronCrown 当前非目标

本文件用于阻止范围膨胀。非目标并不代表永远不做，而是表示在首个垂直切片和核心验证完成前不得进入实施。

## 1. 暂不制作的大型系统

- 玩家建国和完整国家政治。
- 大规模实时 PVP。
- 万人同图或无缝超大世界。
- 跨服战争。
- 海战与完整航海系统。
- 家族、婚姻、继承与多代王朝。
- 复杂宗教、文化与阶层政治模拟。
- 完整自由市场、银行、借贷和金融危机。
- 数百名 NPC 的完整人生模拟。
- 全服语音与复杂社交平台。

## 2. 暂不制作的战斗内容

- 完整第三人称骑砍式动作战斗。
- 骑马物理、复杂碰撞和全身部位伤害。
- 数百单位同时实时模拟。
- 高精度毁城物理。
- 海陆空多军种体系。
- 大量英雄技能与连招。

首版只验证小队编成、半自动战斗、关键指令和可解释战报。

## 3. 暂不制作的经营内容

- 每个村民单独排班。
- 每辆运输车真实物流模拟。
- 数十种人口阶层。
- 数百种资源和商品。
- 复杂税法和官僚体系。
- 手动管理每间工坊的全部配方。

首版资源栏不超过六种核心资源，玩家只决定方向、布局和负责人。

## 4. 暂不制作的商业化内容

- 真实支付。
- 抽卡名将。
- VIP 等级。
- 直接购买城市、爵位和国家职位。
- 付费专属强力兵种。
- 无限加速与无限资源包。
- 额外永久宿将名额。
- 付费避免人物退场。

首个垂直切片只验证玩法，不验证收入。

## 5. 暂不支持的平台与技术

- 首发同时支持 Android、iOS、Windows、macOS 和主机。
- 大规模微服务架构。
- 区块链、NFT 和代币经济。
- 在核心玩法未验证前接入复杂云服务。
- 为未来可能需要而预建全部服务器系统。

开发优先 PC 编辑测试与 Android 验证。

## 6. 禁止伪装成必要基础设施的范围膨胀

以下说法不能自动成为开工理由：

- “以后肯定会用到。”
- “顺手一起做更完整。”
- “模型已经生成了大部分代码。”
- “竞品都有这个功能。”
- “不做就不像大型游戏。”
- “现在重构一次以后会更快。”

新增内容必须说明它如何直接服务当前里程碑的验收指标。

## 7. 重新评估条件

非目标只有在满足以下条件后才可重新进入规划：

1. 首个垂直切片已通过试玩验证。
2. 当前阶段退出标准已经满足。
3. 已建立产品变更 Issue。
4. 已评估人力、技术、内容和维护成本。
5. 已说明不做该功能会阻碍哪个明确指标。
6. 产品负责人罗松涛明确批准。
