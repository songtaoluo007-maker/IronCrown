# IronCrown 首个垂直切片范围

## 1. 目标

用一个 30—60 分钟的可玩版本证明 IronCrown 的产品灵魂，而不是提前制作完整多人 SLG。

必须验证：

1. 普通伙伴能否让玩家产生感情。
2. 从无名者到获得村庄是否有明确成就感。
3. 佣兵或职业武将路线是否具备独立吸引力。
4. 战斗、人物和村庄经营能否形成同一个闭环。

## 2. 玩家完整流程

1. 创建角色并选择一个背景。
2. 进入初始村庄。
3. 认识并招募第一名普通伙伴。
4. 为伙伴设置私人昵称。
5. 完成一次简单贸易或跑腿任务。
6. 接受护送或清剿合同。
7. 招募三名以内核心伙伴和若干普通士兵。
8. 进行第一场半自动战斗。
9. 处理伤员、军饷和补给。
10. 完成第二场难度更高的战斗。
11. 保护一座受到威胁的村庄。
12. 获得村庄临时管理权。
13. 任命一名伙伴担任村长或治安官。
14. 在农业、商业、军事三个方案中选择第一项发展科技。
15. 调整少量村庄建筑或发展方向。
16. 防守一次盗匪袭击。
17. 收到地方领主的册封邀请和佣兵团邀请。
18. 以路线选择悬念结束切片。

## 3. 必须实现的系统

### 3.1 角色与伙伴

- 玩家角色创建。
- 至少三种初始背景。
- 伙伴原名、私人昵称、头像或立绘。
- 武力、智略、治理、交涉四项能力。
- 一个性格、一个天赋、忠诚度和当前职位。
- 招募、同行、任职三种状态。
- 三至五名可招募核心伙伴。
- 伙伴简短个人事件。

### 3.2 小队与补给

- 小队编成。
- 普通士兵与核心伙伴区分。
- 粮食、军饷、士气和伤员。
- 简化装备。
- 行军与任务派遣。

### 3.3 战斗

- 俯视或斜 45 度半自动战斗。
- 至少四类兵种：盾兵、长枪兵、弓兵、骑兵或工程兵中的四类。
- 前排、后排和侧翼编成。
- 冲锋、坚守、集火、撤退四项战术指令。
- 胜败、伤亡、士气和战利品结算。
- 可读战报，说明主要胜负原因。

### 3.4 村庄

- 一个可获得的村庄。
- 粮食、木材、石料、铁材、金币、人口六种核心资源。
- 住宅、农田、仓库、工坊、民兵营和简易防御等 8—12 种建筑。
- 农业、商业、军事三个发展倾向。
- 任命村长或治安官。
- 一个资源短缺事件。
- 一次村庄防守。

### 3.5 科技

- 需求触发。
- 三选一方案。
- 任命研究负责人。
- 时间与资源成本。
- 完成后解锁可见的新能力或建筑。

### 3.6 佣兵合同

首版由 AI 雇主发布：

- 护送合同。
- 清剿合同。
- 驻防或参战合同至少一种。
- 薪资、奖金和伤亡补偿的简化结算。

### 3.7 存档与数据

- 本地存档。
- 角色、伙伴、小队、任务、村庄和科技状态保存。
- 存档版本号。
- 基础迁移入口。
- 可复现随机种子。

## 4. 内容最低量

- 1 个初始村庄。
- 1 个可管理村庄。
- 2—3 个小型地图或场景。
- 3—5 名核心伙伴。
- 3 种背景。
- 4 类兵种。
- 3 种任务类型。
- 2 次主线战斗和若干可选战斗。
- 3 个村庄发展方案。
- 6—9 项科技或发展选项。
- 1 个结尾路线选择。

## 5. 美术范围

推荐：风格化 2.5D 大地图、斜 45 度低多边形战场、高质量人物立绘。

首版允许：

- 使用已授权的 Unity 商店模型和动画。
- 使用 AI 生成的原创人物立绘、头像、背景和 UI 草图。
- 使用模块化建筑。
- 使用统一调色和后期处理弥合不同资产风格。

禁止：

- 使用《权力与纷争》或其他商业游戏提取素材。
- 未确认授权的模型、音乐、字体和图标。
- 为追求写实而引入超出范围的高精度角色管线。

## 6. 技术范围

- Unity + C#。
- PC 编辑与测试，Android 为首个移动目标。
- 数据驱动配置。
- 模块化单体结构。
- 核心逻辑与 Unity 表现层分离。
- 自动化 EditMode 测试，关键战斗与存档增加 PlayMode 测试。
- 不依赖生产服务器。

## 7. 验收标准

垂直切片必须：

- 从新游戏到结尾可完整通关，无阻断性 Bug。
- 伙伴昵称、能力、关系和职位能正确保存。
- 战斗结果可解释、可复现。
- 村庄资源不会出现明显负数、重复结算和无限刷取。
- 任命不同伙伴能产生可感知差异。
- 玩家能在一小时内理解领主与佣兵两条未来路线。
- Android 中端设备目标 30 FPS，核心交互无明显卡顿。
- 所有第三方资产有来源与许可记录。

## 8. 退出标准

满足以下条件后，才允许进入小规模联机设计：

- 至少 20 名目标玩家完成试玩。
- 超过一半测试者能说出一名印象深刻的伙伴。
- 超过一半测试者认为获得村庄是明显身份跃迁。
- 领主和佣兵路线均有人主动选择。
- 核心流程无高频阻断 Bug。
- 产品负责人确认战斗和伙伴系统具有继续开发价值。

未满足时优先修改核心循环，不得以增加国家、跨服、抽卡或新玩法掩盖问题。
